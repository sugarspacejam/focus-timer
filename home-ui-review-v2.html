Why this solves the “many tasks” problem

Your instinct is right: a pure vertical screen becomes hostile when the user has lots of tasks. A paying user won’t tolerate a UI that punishes them for relying on it.

This design keeps the home surface stable:

  • Home stays short: stats + quick start + 3–6 recent tasks.
  • Task Library scales: 10 or 200 tasks doesn’t bloat home.
  • No calendar needed: retrieval is driven by behavior (recent/pinned) instead of dates.
  • Management is contained: edit/delete are not screaming at the user on every row all day.

Micro-annoyances this avoids

Paying users open the app in low-energy moments. Tiny friction breaks the spell:

  • Scrolling past settings and stats every time just to find a task.
  • Repeated Start/Edit/Delete controls on every row causing visual noise.
  • The home screen reminding the user of everything they’re not doing.

Alternate 1: Tabs (Today / Tasks / Settings)

Same principles, more native information architecture. Best if you expect heavy daily use and want a clean separation between launching and managing.
